# Contributing (GitBook Content)

## Structure

- Use category folders in lowercase kebab-case.
- Put topic files under the correct category folder.
- Keep root navigation aligned with `SUMMARY.md`.

## Topic File Rules

- Include YAML frontmatter.
- Include exactly one `## Related Topics` section near the top.
- Prefer relative Markdown links for internal references.
- Keep titles clear and publish-oriented.

## GitBook Readiness Checklist

- Update category `README.md` when adding/removing topics.
- Update `SUMMARY.md` for sidebar navigation.
- Ensure no broken internal links.
- Keep naming convention consistent (`topic-slug.md`).
